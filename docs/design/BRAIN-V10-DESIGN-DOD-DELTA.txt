## Why I am choosing these features over the competing ones

### 1) Merged PreflightAssessment beats split Gate 1 + CS Audit

I keep the V3.0 merged preflight. The live architecture already runs a single `run_preflight(...)` stage, returns `NEED_MORE` from there, stores `proof.preflight`, and carries no real separate admission owner in the pipeline. V3.0B’s split Gate 1 / CS Audit is a structural fork, not a delta. The lingering `gate1` names in `BrainResult` and `proof.py` are legacy residue to clean up, not evidence that the split architecture should return.    

### 2) Keep ANALYSIS A1–A7; reject the A1–A3 simplification

The current DOD and code already support ANALYSIS `A1–A7`, including artifact-integrity checks, evidence floor, dimension floor, and total-argument floor. V3.0B’s `A1–A3` version throws away guardrails that already exist and are already useful. New 20 is right here.    

### 3) Defer canonical entity IDs

Report.md is stronger than new 20 on this point. Canonical IDs touch prompts, trackers, decisive-claim extraction, contradiction linkage, synthesis, and proof schema all at once. That is not a clean V3.1 delta on top of the current codebase. The current code already has claim IDs, argument IDs, contradiction IDs, and blocker IDs where needed; the real gap is not total absence of IDs, but incomplete cross-round normalization. That should be a V3.2 structural program, not a V3.1 delta.   

### 4) Defer full claim-aware pinning

New 20 correctly identifies the downstream-consumer problem: if pinning lands, every consumer that iterates active evidence must become pin-aware. That insight is good. But that is exactly why the feature is too invasive for this delta. Report.md is stronger on the release decision: keep the two-tier ledger and add a forensic eviction overlay now; defer full pinning until the entity/linkage layer is mature.   

### 5) Defer Virtual Frames

Report.md makes the best positive case for Virtual Frames, but new 20 makes the better release call. V3.0 already has `DimensionSeeder`, `PerspectiveCards`, fixed adversarial Kimi in R1, and the exploration-stress trigger. For V3.1, **Breadth Recovery + Anti-Groupthink Search** address the concrete premature-convergence failure with less new controller state and no new pre-R1 synthetic frame lifecycle.   

### 6) Keep D-rules explicit, but do not keep D1–D17 exactly as written in new 20

New 20 is right that the DECIDE contract should be one explicit rule table. But its `D15` depends on pin-cap enforcement, and I am deferring pinning. So the optimal V3.1 compromise is **explicit D1–D16**: add residue-threshold escalation and suspicious-agreement escalation, keep groupthink escalation, and make final DECIDE the last rule. That is more internally coherent than pretending a deferred feature still owns a live Gate 2 rule.
